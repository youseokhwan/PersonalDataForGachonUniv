<파이썬 멘토링>

1. Intro
  1) 간략한 멘토소개
  2) 멘티들 학과, 학년, 희망 직군, 언어 경력(없으면 말고) 물어보기
  3) 학교 수업으로 파이썬 듣는 사람 있는지 파악(IDE, 기본서 등)

2. 멘토링 목표
  1) 파이썬 기초적인 문법 숙달
  2) 언어 공부 노하우, Tip
    (1) 언어 처음 공부 시 의존성 문제(약간 문과 스타일 공부 지양)
        - 한 번에 이해 안돼도 일단 넘어가기
    (2) 문법은 달달 외우지말고 필요할 때마다 구글링
        - 실제 구글링하는 것 자주 보여주면서 설명
    (3) 주먹구구식 코드 금지
        - 플로우 대충이라도 머릿 속에 구상해놓고 타이핑 시작할 것
    (4) 기타
        - 네이밍
        - 용어, 개념 부분에서 한글, 영어 동시에 숙지
  3) GitHub 소개, 해야하는 이유(취업 관련)
    (1) 1일 1커밋(하루10분, 주5회이상)
       * 힘들 것 같은 분위기면 그날 그날의 스터디 내용만이라도 정리해서 커밋하게하기
    &&&&&&&&&& 내용 추가(GitHub란? / GitHub를 해야하는 이유, 인크루팅(포트폴리오 등)에서 GitHub의 활용도 &&&&&&&&&&

3. 주요 언어(C, C++, JAVA, Python 등)의 패러다임
    * 최대한 아주 간략하고 쉽게 전문용어 다 빼고 설명
    * 게임 예시, 최댓값 구하는 수학 함수들 등 예시로 들어서 모듈(OOP)등 설명

4. 파이썬3 소개
  1) 간단한 역사
    - 귀도 반 로섬(spam, eggs, 42 등)
    - 2 vs 3 등
    - &&&&&&& 내용 추가 ^&&&&&&&&&&&

5. 스터디 구체적인 진행(커리큘럼)
  1) 파이썬 기본서의 목차 참고하여 문법은 그 순서대로 진행하되 간단한 것만
  2) 기본 문법 설명 -> 예시(활용) 직접 보여주고 -> 간단한 문제풀이 시키기
  3) 모르는 용어/개념 적어놨다가 GitHub에 정리
