--- 7장 File Permission ---

0. 개요

1. 파일의 속성
- "ls -l" 명령어로 파일의 속성 값 확인
- '-'는 일반 파일, 'd'는 디렉토리
- "file" : 파일의 종류를 알려주는 명령
- "groups" : 사용자가 속한 그룹을 알려주는 명령

Q. 접근 권한의 종류 3가지를 서술하시오.
1) 파일
(1) 읽기 : 파일을 읽거나 복사할 수 있음
(2) 쓰기 : 파일을 수정, 이동, 삭제할 수 있음
(3) 실행 : 파일을 실행할 수 있음
2) 디렉토리
(1) 읽기 : "ls" 명령으로 디렉토리를 볼 수 있음
(2) 쓰기 : 파일을 생성하거나 삭제할 수 있음
(3) 실행 : "cd" 명령을 사용할 수 있음, 파일을 디렉토리로 이동하거나 복사할 수 있음

2. 파일의 접근 권한
Q. 접근 권한의 표기 방법에 대해 서술하시오.
1) 읽기 권한은 r, 쓰기 권한은 w, 실행 권한은 x로 표시(없으면 -로 표시)
2) 소유자, 그룹, 기타 사용자 순으로 묶어서 표시
- rw-r--r--

- "chmod" 명령으로 접근 권한 변경

3. 기호를 이용한 파일 접근 권한 변경
- 파일 소유자(u), 그룹(g), 기타 사용자(o), 전체 사용자(a)
- 권한 부여(+), 제거(-), 설정(=)
- 읽기(r), 쓰기(w), 실행(x)
- u+x : 소유자에게 실행 권한 부여
- o-r : 기타 사용자의 읽기 권한 제거
- +wx : 전체 사용자에게 읽기, 실행 권한 부여
- u=rwx : 소유자에게 읽기, 쓰기, 실행 권한 부여
- go+w : 그룹 사용자와 기타 사용자에게 쓰기 권한 부여
- "chmod u-w test.txt"

4. 숫자를 이용한 파일 접근 권한 변경
- rwxrwxrwx -> 777
- rwxr-xr-x -> 755 등
- "chmod 444 test.txt" : r--r--r--로 변경됨

Q. 파일 접근 권한을 변경하는 2가지 방법을 서술하시오.
1) 기호를 이용한 변경
2) 숫자를 이용한 변경

5. 기본 접근 권한 설정
- 일반 파일 : 644
- 디렉토리 : 775
- "umask" : 마스크 값 확인
- "umask 077" : 마스크 값을 077로 변경

- 마스크 값 : 파일이나 디렉토리 생성 시 부여하지 않을 권한을 지정해놓는 것
- 최초 생성 시 기본 생성 권한에서 마스크 값을 뺀 값이 부여 됨

6. 특수 접근 권한
1) SetUID(4000)
- 실행 시 사용자가 아닌 소유자의 권한을 부여
- 실행 파일에만 설정 가능
- 소유자 부분에서 x 대신 s로 표기 (rwSr-xr-x)
- 대표적으로 "passwd" 명령어

// 기존에 실행 권한이 없으면 S(대문자)
// 있으면 s(소문자)로 표시된다.
// 실행 파일은 S, 디렉토리는 s?

Q. /usr/bin/passwd 파일과 /etc/passwd 파일의 차이점은 무엇인가?

2) SetGID(2000)
- 실행 시 그룹 소유자의 권한을 부여
- 실행 파일 및 디렉토리에도 설정 가능
- 그룹 부분에서 x 대신 S로 표기 (rw-r-Sr-x)
- 하위 디렉토리에도 적용되는데, 적용 이후에 만드는 것부터 적용된다.

3) Sticky bit(1000)
- 모든 사용자는 읽고 쓰기가 가능하나 삭제는 소유자만 할 수 있음
- 디렉토리에 설정 가능
- 기타 사용자 부분에서 x 대신 t로 표기 (dwxrwxrwt)
- 대표적으로 /tmp, /var/tmp
