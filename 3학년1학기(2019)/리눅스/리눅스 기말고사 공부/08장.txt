0. 개요

1. 프로세스의 개념
- 프로세스 : 현재 시스템에서 실행 중인 프로그램
- PID(Process ID) : 각 프로세스의 고유한 번호

3) 데몬 프로세스 : 특정 서비스를 제공하기 위해 존재하며 리눅스 커널에 의해 실행
4) 고아 프로세스 : 부모 프로세스가 먼저 종료된 자식 프로세스, 1번 프로세스가 새로운 부모 프로세스가 됨
5) 좀비 프로세스 : 자식 프로세스가 종료했는데도 테이블에 남아있는 경우

2. 프로세스 관리 명령
- "ps" : 프로세스 목록을 보는 명령
- "ps -f" : 상세 정보 보기(UID, PID, PPID, TIME 등)
- "ps a" : 터미널에서 실행한 프로세스 정보 출력
- "ps au" : 터미널에서 실행한 프로세스의 상세 정보 출력

Q. a 옵션과 au 옵션에서 사용되는 문자들의 의미? (설마)

- "ps -e" : 전체 프로세스 목록 출력
- "ps -ef" : 전체 프로세스의 상세 정보 출력
- "ps -u" : 특정 사용자의 프로세스 목록 출력
- "ps -p" : 특정 프로세스의 정보 출력
- "pgrep" : ps | grep

- "kill" : "ps -ef | grep man"하여 PID 확인 후 "kill -9 PID"로 프로세스 종료
- "pkill" : 프로세스 이름으로 종료 "pkill man"
- "top" : 실행중인 프로세스 정보를 주기적으로 출력

Q. 작업 제어 3가지를 서술하시오
1) 작업 전환 : 포그라운드 <-> 백그라운드 전환
2) 작업 일시 중지
3) 작업 종료

- "jobs" : 백그라운드 작업을 모두 출력
- Ctrl + Z : 포그라운드 작업 일시 중지
- "bg %1" : 1번 작업을 백그라운드로 전환
- "fg %1" : 1번 작업을 포그라운드로 전환
- Ctrl + C : 포그라운드 작업 종료
- "kill %1" : 1번 백그라운드 작업 종료
- "nohup" : 로그아웃한 뒤에도 백그라운드 작업 계속 실행

4. 작업 예약
Q. 작업 예약할 수 있는 2가지 방법을 서술하시오.
1) 정해진 시간에 한 번 실행 (at)
  - 시간 지정 -> "at 10am Jul 31"
  - 실행 결과는 메일로 전달(/var/spool/mail/user1)
  - "at -l", "atq" -> 예약 목록 확인
  - "at -d 7", "atrm 7" -> 7번 작업의 예약 삭제
2) 정해진 시간에 반복 수행 (crontab)
  - 예시) "30 23 1 * * /usr/bin/ls -l ~user1 > ~user1/cron.out"
  - 분, 시, 일, 월, 요일, 작업 내용 순
  - "crontab -e", "vi /etc/crontab" : crontab 파일 생성 및 편집(vi)
  - "crontab -l" : crontab 내용 확인
  - "crontab -r" : 예약 작업 삭제(전체인지 첫 줄인지?)
  - 작업 후 "systemctl restart crond" 필수

Q. at 명령의 사용을 제한하는 방법을 서술하시오.
- /etc/at.allow 파일에 있는 사용자만 at 명령 사용 가능(deny 무시)
- /etc/at.deny 파일에 있는 사용자를 제외한 모든 사용자가 사용 가능
- 둘 다 없으면 root만 사용 가능
- 디폴트 값 : /etc/at.deny를 빈 파일로 두면 모든 사용자가 at 명령 사용 가능
