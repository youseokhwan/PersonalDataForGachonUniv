--- 10장 Software Management ---

0. 개요

1. RPM 패키지 설치
- RPM(Redhat Package Manager) : 레드햇에서 만든 패키지 관리 도구

Q. RPM의 특징에 대해 서술하시오.
1) 바이너리 파일로 되어있어 컴파일 필요없음
2) 패키지의 파일들이 디렉토리로 바로 설치
3) 한 번 설치된 패키지 파일들을 일괄적으로 삭제할 수 있음
4) 기존 설치된 패키지를 삭제하지 않고 업그레이드 가능
5) 의존성 : 관련 패키지가 먼저 설치되어야 설치 가능

- "rpm --version"
- http://rpm.pbone.net에서 검색하여 다운로드
- "rpm -i mc-4.8.7~" : 패키지 설치(Uvh를 더 많이 사용)
- "rpm -Uvh mc-4.8.7~" : Upgrade, Verbose, Hashtag
- "rpm -qa | grep bash" : 'bash'가 포함된 패키지 목록 출력
- "rpm -qf /bin/ls" : ls 파일 포함한 패키지 목록 검색(절대경로)
- "rpm -qi bash" : bash 패키지 상세 정보 출력
- "rpm -qR bash" : bash 패키지 의존성 확인
- "rpm -e mc" : mc 패키지 삭제

2. yum을 이용한 패키지 설치
- yum : Yellowdog Updater Modified

Q. yum의 특징을 서술하시오.
1) 의존성 문제를 해결하고 원격 자동 업데이트와 설치 가능
2) 설치 가능한 패키지의 정보가 담긴 Repository를 가지고 있음

Q. yum의 작동 방식을 서술하시오.
- 인터넷 연결 필수
1) yum install 입력
2) /etc/yum.repos.d/의 repo 파일을 열어서 URL 확인
3) 전체 패키지 목록 요청
4) 전체 패키지 목록 파일만 다운로드
5) 관련된 패키지들을 화면에 출력
6) y를 입력하여 설치 요청
7) 다운로드하여 설치

Q. yum의 설정 파일 3가지를 서술하시오
1) /etc/yum.conf : 변경 필요 없음
2) /etc/yum.repos.d/ : 네트워크의 주소가 들어 있는 파일들
3) /etc/yum.repos.d/*.repo : 패키지 파일들

- "yum list available" : 설치 가능한 패키지 목록 출력
- "yum list updates", "yum check-update" : 업데이트 가능한 패키지 목록 출력
- "yum list installed" : 설치되어 있는 패키지 목록 출력
- "yum info autotrace" : autotrace 패키지의 정보 검색
- "yum remove autotrace" : autotrace 패키지 삭제

3. 파일 아카이브와 압축
- 아카이브 : 파일을 묶어서 하나로 만든 것
- tar(Tape ARchive) : 여러 파일을 아카이브로 묶어 테이프와 같은 이동식디스크에 저장하기 위한 명령
- "tar cvf ch10.tar 다운로드" : 다운로드 디렉토리의 파일들을 ch10.tar 아카이브로 생성
- "tar tvf ch10.tar" : 아카이브 내용 확인
- "tar xvf ch10.tar" : 아카이브 풀기

Q. 파일을 압축하는 방법 2가지를 설명하시오.
1) gzip
  - "tar cvfz ch10.tar.gz ch10"
  - "gzip example.tar" -> example.tar.gz
  - "zcat example.tar.gz" : .gz 파일 내용 보기
  - "gunzip example.tar.gz" : .gz 압축 풀기
2) bzip2
  - "tar cvfj ch10-1.tar.bz2 ch10"
  - "bzip2 bziptest.tar" -> bziptest.tar.bz2
  - "bzcat bziptest.tar.bz2" : .bz2 파일 내용 보기
  - "bunzip2 bziptest.tar.bz2" : .bz2 압축 풀기
3) gzip이 압축률이 더 좋음

4. 소프트웨어 컴파일
- gcc : 리눅스 C 컴파일러
- "gcc hello.c" -> a.out으로 실행파일 생성
- "gcc -o test hello.c" -> 이름 지정해서 실행파일 생성

Q. 소스를 받아 설치하는 과정을 설명하시오
1) 소스코드 다운(test.tar.gz)
2) 압축 풀기(gunzip test.tar.gz)
3) 설치문서 읽기
  - configure : 필요한 도구와 라이브러리
  - make : 소스코드 컴파일
  - make install : 설치
4) make uninstall : 제거
