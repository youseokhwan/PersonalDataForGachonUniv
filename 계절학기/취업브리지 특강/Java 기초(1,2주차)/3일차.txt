19.08.08(목) 지브릿지 3일차

- Class
- 가변 인자(Variable arguments) : 인자의 개수가 유동적인 메소드

- 자바 메모리 관리는 3영역으로 구분
  1 메서드 영역(클래스 영역) : 클래스 변수, static 메서드 등
  2 힙 영역 : 대부분의 메모리, 인스턴스
  3 스택 영역(호출 스택) : 메서드 실행에 필요한 메모리 공간(지역변수, 매개변수 등)

- 반복문
  for(변수 : 집합형데이터)
  :는 in으로 읽음, 모든 데이터를 순회하는 것.
  시작과 끝을 지정해야하면 기존 for문 사용

- 배열
  실제로 배열을 사용하기엔 효율이 좋지 않음
  배열보다는 List를 주로 사용
  asList() : 배열을 List로 변환
  toString() : Object의 toString()을 오버라이딩 (배열의 내용 출력)
  
- 디버거 사용
  중단점(break point) 더블클릭해서 지정하고, Debug Loop(벌레모양) 아이콘 클릭하여 디버거 실행
  Step In, Step Over

- import 할 때 *로 모두 지정하는 것은 권장되지 않음

- 최근엔 로깅 프레임워크로 logback을 많이 사용하는 추세
- Logger의 경우 Singleton으로 new를 사용하지 않는다. (인스턴스를 따로 생성하지 않는다)
- 모든 java파일엔 Logger logger = Logger.getLogger("MyLogger"); 문구가 들어간다.
  이 또한 반복되는 작업 -> 스프링프레임워크에선 하지 않아도 됨, AOP
