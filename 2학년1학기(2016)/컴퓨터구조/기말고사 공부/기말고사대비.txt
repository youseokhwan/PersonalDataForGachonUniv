마이크로명령어: 마이크로연산을 나타내는 2진 비트 표현

순서제어: 다음에 실행될 명령어의 순서를 제어

수직적 마이크로프로그래밍: 최소의 비트들만 두고, 필요할 때마다 확장시키는 방식
수평적 마이크로프로그래밍: 필요한 만큼 비트들을 두고, 직접 사용하는 방식

액세스: 기억장치에 대한 읽기 및 쓰기 동작
직접 액세스: 정보의 위치로 직접 이동시킨 후에, 순차적 검색으로 탐색
임의 액세스: 정보의 위치를 임의로 이동, 걸리는 시간 같음
연관 액세스: 각 기억 장치의 키값으로 탐색

휘발성 기억장치: 전원이 중단되면 기억장치에 적재된 데이터가 모두 사라지는 기억장치
비휘발성 기억장치: 전원이 중단되어도 데이터가 사라지지않는 기억장치

계층적 기억장치시스템: 속도, 가격 등 다양한 기억장치들을 계층적으로 설치한 시스템, 가성비 좋아짐

지역성의 원리: CPU가 한정된 영역을 집중적으로 탐색하게 되는 것

내부 기억장치: CPU가 직접 액세스할 수 있는 장치 (CPU 레지스터, 캐시, 주기억장치)
외부 기억장치: CPU가 직접 액세스할 수 없고, 독립적으로 액세스하는 장치 (보조기억장치, 디스크 캐시, CD-ROM등)

캐시 메모리:CPU와 주기억장치 간의 속도차이 문제를 해결하기 위한 중간 기억장치
디스크 캐시: 주기억장치와 보조기억장치 간의 속도차이 문제를 해결하기 위한 중간 기억장치

RAM:Random Access Memory : 읽기와 쓰기가 모두 가능한 기억장치
DRAM: Dynamic RAM: 충전 방식을 이용한 RAM, 주기적 충전해야함
SRAM: Static RAM: 플립플롭 방식을 이용한 RAM
RAS:: Row 행 주소 래치 신호
CAS:열 주소 래치 신호

ROM: Read Only Memory: 저장된 내용을 읽기만 할 수 있는 기억장치
PROM: 한 번은 수정할 수 있는 ROM
EPROM: 자외선을 이용하여 여러번 수정할 수 있는 ROM
EEPROM: 전기를 이용하여 여러번 수정할 수 있는 ROM

캐시 적중: 액세스 하려는 데이터가 이미 적재된 상태
캐시 미스: 액세스 하려는 데이터가 없어서 인출해와야하는 상태

요구 인출: 요구한 데이터만을 인출해오는 방식
선인출: 요구한 데이터와 인접한 데이터들까지 인출해오는 방식

사상 방식: 데이터들이 어느 캐시 라인에 적재될지 결정하는 알고리즘
직접 사상: 데이터가 어느 한 라인에만 적재될 수 있는 방식
완전-연관 사상: 데이터가 어느 라인으로든 적재될 수 있는 방식
세트-연관 사상: 데이터가 어느 한 세트로만 적재될 수 있는 방식

최소 최근 사용 LRU: 최근에 사용된 것 중 가장 빈도가 적은 데이터를 삭제
최소 사용 빈도 LFU: 기억장치에 적재된 이후 가장 적게 사용한 데이터를 삭제
FIFO: 먼저 기억된 데이터를 먼저 삭제
임의: 임의로 삭제

쓰기 정책:데이터를 쓸때 시기와 방법을 정하는 방식
write-through :캐시에 쓰기 동작을 수행할 때 주기억장치에도 동시에 진행하는 방식
write-back:캐시에만 쓰기 동작을 수행하는 방식

동기식 DRAM (SDRAM):각종 작업이 시스템 클록 주기에 동기화되어 진행되는 DRAM

DDR: Double Date Rate: 데이터를 클록당 두번 전송하는 기법 (상승, 하강)

SIMM:단면 모듈: 단면에만 칩이 있는 PCB
DIMM:양면 모듈: 양면에 칩이 있는 PCB	

PRAM:화합물반도체를 이용하여 2진 정보를 저장하는 RAM
FRAM:강유전체를 이용하여 2진 정보를 저장하는 RAM
MRAM:강자성체를 이용하여 2진 정보를 저장하는 RAM

이동헤드 디스크:하나의 헤드로만 이루어진 디스크 팔로 데이터를 액세스하는 디스크
다중헤드 디스크:여러개의 헤드로 이루어진 디스크 팔로 데이터를 액세스하는 디스크
고정헤드 디스크:각각 헤드를 장착함으로써 움직일 필요가 없는 디스크

등각속도: 중심부에 가까운 트랙을 액세스하든 먼 트랙을 액세스하든 속도가 같음

탐색시간:헤드를 액세스할 트랙으로 이동시키는 시간
회전 지연시간:해당 트랙에 도착한 이후, 원하는 섹터로 회전하는 시간
데이터 전송시간:헤드를 통하여 섹터들을 전송하는데 걸리는 시간

RAID: 디스크들을 배열로 구현하여 높은 신뢰성을 보장하는 디스크시스템

디스크 인터리빙: 데이터를 여러 디스크들에 분산 저장하는 방식
MTTF:결함이 발생하는 평균 시간
검사 디스크: 오류를 검사하기 위해 마련된 별도의 공간
MTTR: 결함을 수정하는데 걸리는 시간

작은 쓰기 문제: RAID-5에서 액세스할때 네번의 디스크액세스가 필요하게되는 문제점

SSD: solid state drive: 플래시 메모리칩들을 이용한 저장장치
플래시 메모리:EEPROM의 일종, 저장밀도 높고 신뢰성 높은 저장장치

SLC:한 비트씩 저장하는 플래시 메모리 칩
MLC:두 비트씩 저장하는 플래시 메모리 칩
TLC:세 비트씩 저장하는 플래시 메모리 칩

플래시 변환 계층 FTL:플래시 메모리의 구조를 HDD의 구조로 변환해주는 S/W
마모 평준화:페이지를 평준화해서 수명을 연장시키는 기술
쓰레기 수집:무효 페이지를 한번에 삭제하는 기술
초과 대비공간:마모 평준화를 위해 SSD에서 마련한 별도의 공간

CD-ROM:데이터를 읽기만 가능한 광디스크
등선속도: 헤드의 속도가 트랙의 위치에 상관없이 일정하도록하는 방식

CD-R: 한번은 수정이 가능한 CD-ROM
CD-RW: 여러번 수정이 가능한 CD-ROM

블루-레이 디스크 BD:짧은 파장의 레이저를 이용한 광디스크

BD-R:한번은 수정이 가능한 BD
BD-RE: 여러번 수정이 가능한 BD


버스 중재:


동기식 버스: 버스 클록을 기준으로 동기화하는 버스
비동기식 버스: 버스 클록을 사용하지않는 버스

병렬 중재 방식: 각각 독립적으로 요구 신호를 보내고 처리하는 방식
직렬 중재 방식: 각 신호선이 직렬로 연결되어있는 방식
중앙집중식 중재 방식: 중앙에서 한번에 처리하는 방식
분산식 중재 방식:별도의 버스마스터에 의해 처리되는 방식

버스 요구신호 BREQ: 버스 사용을 요청하는 신호
버스 승인신호 BGNT:버스 사용을 승인하는 신호

하드웨어 폴링 방식:하드웨어를 이용한 주기적 검사를 이용하여 중재기능을 수행하는 방식
소프트웨어 폴링 방식:소프트웨어를 이용한 프로그램을 통해 중재하는 방식

I/O 제어기:
프로그램을 이용한 I/O: CPU가 반복적으로 I/O 신호를 처리하는 것

기억장치-사상 I/O: 기억장치와 같은 방법으로 액세스
분리형 I/O:각각 별도로 액세스
인터럽트-구동 I/O:

다중 인터럽트 방식:요구 신호선과, 승인 신호선이 한개씩 존재
데이지 체인 방식:모든 제어기들은 하나의 신호선을 공유
소프트웨어 폴링 방식:한개의 신호선이 CPU와 모든 제어기들에 연결

직접기억장치액세스(DMA) :CPU개입없이 독립적으로 수행하는 매커니즘

병렬처리:다수의 프로세서들이 여러 프로그램을 분담하여 처리하는 기술
멀티-스레딩:스레드를 여러 프로세서들이 처리하는 기술

명령어 스트림:프로세서로 들어오는 명령어들의 흐름
데이터 스트림:프로세서로 들어오는 데이터들의 흐름

SISD:단일 명령어 스트림-단일 데이터 스트림 시스템
SIMD:단일 명령어
